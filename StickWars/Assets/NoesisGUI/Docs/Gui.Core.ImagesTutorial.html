Using Images

The Image control is used to insert an image whose location is specified in the Source attribute:

<Image Source="Images/melon1.png"/>

NOTE: Paths are relative to the location of the xaml file

As the rest of controls, Image is measured and arranged, so its size depends not only on the original image dimensions. Layout properties determine the space available for the Image, and its Stretch property defines how the image fills that available space:

  • None: dimensions are preserved.
  • Fill: the image fills the available space, dimensions are modified.
  • Uniform: the image fills as much as possible the available space preserving the aspect ratio.
  • UniformToFill: the image totally fills the available space preserving the aspect ratio and clipping the image if it is too large.

ImageBrush

ImageBrush uses an image to paint an area of the graphics objects. It is normally applied to the Background property of a Panel or Border, or to Fill a Shape. The image is specified in the ImageSource property.

<Grid
  xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
  xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Grid.Background>
        <ImageBrush ImageSource="Images/melon1.png"/>
    </Grid.Background>
</Grid>

The ImageBrush derives from the TileBrush class. When you paint an area by using a TileBrush, you use three components: content, tiles, and the output area.

TileBrush defines some interesting properties that configure how tiles are placed in the output area.

Viewport

This property specifies the position and dimensions of the tile in the output area. By default, a TileBrush has a single tile that fills the whole output area.

We use the ViewportUnits property to specify whether the Viewport uses absolute or relative (by default) coordinates. If the coordinates are relative, they are relative to the size of the output area. The point (0,0) represents the upper-left corner of the output area, and (1,1) represents the lower-right corner of the output area.

TileMode

The TileMode property allows you to specify how the content of a TileBrush is repeated to fill the output area. To create a pattern, you set the TileMode to Tile, FlipX, FlipY, or FlipXY and you must also set the Viewport of the TileBrush so that it is smaller than the area that you are painting; otherwise, only a single tile is produced, regardless which TileMode setting you use.

Texture Atlas

Although you can use the tool you wish, we provide a plugin for TexturePacker that directly generates a valid xaml that can be used as an atlas in NoesisGui. The plugin is located under Gui/3rdParty/Content/TexturePacker. The following is an atlas generated with that tool:
