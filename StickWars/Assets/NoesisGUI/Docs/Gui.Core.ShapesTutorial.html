Rendering shapes

Tutorial Data

In this tutorial we are going to learn how to work with Shapes and Brushes.

Shape elements are used for drawing vector graphics. A Shape has a Fill property, which is the background color, and a Stroke property, which is the outline color. Both properties are of type Brush. With the StrokeWidth property you can control the thickness of the outline.

NoesisGUI supports the following basic shapes:

  • Ellipse: for drawing oval or circles
  • Rectangle: for rectangles or squares with or without rounded corners
  • Line: for segments connecting two points
  • Path: for complex shapes

Stroke and Fill properties need to be set to a valid brush:

  • Solid: for example "Red" or "Blue" or #aarrggbb for alpha, red, green and blue components respectively.
  • Linear gradient: for painting a gradient along a line specified by StartPoint and EndPoint properties.
  • Radial gradient: for gradients along a circle.
  • Image: for painting with a image

Shapes are typically placed inside a Canvas because it supports absolute positioning of its child objects. The following example illustrates some of the basic shapes:

<Canvas Height="300" Width="300" Background="LightGray">

 <Line X1="30" Y1="30" X2="130" Y2="130" Stroke="Black" StrokeThickness="8"
        StrokeStartLineCap="Round" StrokeEndLineCap="Round"/>

 <Ellipse Canvas.Top="25" Canvas.Left="160" Height="120" Width="120"
        StrokeThickness="4" Stroke="Black">
  <Ellipse.Fill>
   <RadialGradientBrush GradientOrigin="0.75,0.25">
     <GradientStop Color="Yellow" Offset="0.0" />
     <GradientStop Color="Orange" Offset="0.5" />
     <GradientStop Color="Red" Offset="1.0" />
   </RadialGradientBrush>
  </Ellipse.Fill>
 </Ellipse>

 <Rectangle Canvas.Left="45" Canvas.Top="250" Height="150" Width="80" RadiusX="9" RadiusY="9"
        Fill="LightBlue" StrokeThickness="2" Stroke="Green" StrokeDashArray="3,1">
  <Rectangle.RenderTransform>
    <RotateTransform Angle="-80" />
  </Rectangle.RenderTransform>
 </Rectangle>

</Canvas>

Apart from the details already commented above there are the following interesting details in this example:

  • The start and end stroking caps are being changed through the StrokeStartLineCap and StrokeEndLineCap properties.
  • Stroke dashing is set through the StrokeDashArray property.
  • Elements can define their position inside the Canvas through attached properties or changing the RenderTrasform property.

Paths are more complex shapes used to draw figures expressed in a special path markup syntax, a mini-language for making compact code. More information about this language can be found in the SVG specification. Inkscape is an amazing tool for converting from SVG files to XAML path format.

<Canvas Height="440" Width="385">

 <Path Fill="#FFF2F000" Stroke="#FF000000" StrokeThickness="9"
  Data="m 608.07798 229.66418 c -88.45345 78.6253 -223.89753 70.658 -302.52283 -17.79545
        -78.62529 -88.45345 -70.65799 -223.897534 17.79546 -302.522828 86.89794 -77.242622
        219.53868 -71.098432 298.92375 13.846751 L 465.71429 69.505043 z">
  <Path.RenderTransform>
   <TranslateTransform X="-245.7143" Y="150.8571"/>
  </Path.RenderTransform>
 </Path>

 <Ellipse Height="45" Width="45" Fill="#FF000000">
  <Ellipse.RenderTransform>
   <TranslateTransform X="201" Y="110"/>
  </Ellipse.RenderTransform>
 </Ellipse>

</Canvas>
